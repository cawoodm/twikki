# Mobile Responsive & PWA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make TWikki usable on phones and tablets — fix the capped-search bug, viewport/layout fundamentals, touch interaction, content overflow, and PWA status-bar polish.

**Architecture:** Almost entirely CSS, concentrated in two theme files (`$CoreThemeLayout.css` = structure/responsive, `$CoreThemeAppearance.css` = sizes/touch) plus a handful of small edits to `$BaseVariables.css`, `$MainLayout.html`, `index.html`, and three plugin CSS files. No platform or plugin _logic_ changes — the drawer scrim is pure HTML+CSS so the no-plugin invariant holds. Verified behaviours are tested with a new Playwright mobile/touch spec; non-observable declarations (dvh, safe-area insets, viewport metas, touch-action) are guarded by a new source-grep unit test.

**Tech Stack:** Plain CSS (cascade layers — see CLAUDE.md "Theming"), HTML, Playwright e2e (Chromium device emulation), Node `node:test` unit tests.

## Global Constraints

- **No CSS variable without a default.** Every `var(--x)` referenced anywhere under `src/` must be declared in `src/modules/core.defaults/$BaseVariables.css` — enforced by `test/unit/tokens.test.js`. If a task introduces a new token, add its default in the same task. Prefer literal values over new tokens unless the value is reused.
- **No ESM in modules/packages.** Files under `src/modules/` and `src/packages/<pkg>/` are plain IIFEs/scripts — never add `import`/`export`. (This plan adds no JS to those dirs anyway.)
- **No-plugin invariant.** Core must stand alone with zero plugins. The drawer scrim must be HTML+CSS only (no plugin/JS dependency).
- **Prettier style** for any JS/JSON touched: single quotes, `bracketSpacing:false` (`{x}` not `{ x }`), `arrowParens:avoid`, `trailingComma:all`.
- **Source files only.** Edit sources under `src/`; never edit `public/packages/*` or `public/modules/*` (gitignored build artifacts regenerated by the Vite plugin / `npm run compile`).
- **Run e2e with:** `npx playwright test test/e2e/responsive.spec.js` (the config auto-starts the dev server on port 5180). Run unit tests with `node --test test/unit/<file>.test.js`.
- **Breakpoint vocabulary:** phone `≤600px` (existing), tablet `601px–1024px` (new). Touch detection via `@media (pointer: coarse)` / `@media (hover: none)`.

### Task 1: P0 — full-width mobile search overlay (#1, #14)

The reported bug. `#search-results` is `position:absolute; left:0; right:0` inside `#explorer-search`, which lives in the `min(85vw,280px)` drawer (`$CoreThemeLayout.css:36-45`, `:169-179`) — so results are capped at ~280px and trapped in the drawer. Fix: on `≤600px` render results as a full-width fixed bottom-sheet using `dvh` (a bottom sheet sidesteps brittle anchor-offset math and is the standard mobile results pattern). Desktop behaviour is untouched.

**Files:**

- Create: `test/e2e/responsive.spec.js`
- Modify: `src/modules/core.defaults/$CoreThemeLayout.css` (append a rule inside the existing `@media (max-width: 600px)` block, currently `:160-189`)

**Interfaces:**

- Consumes: existing `bootApp`, `createTiddler` from `test/e2e/helpers.js`.
- Produces: `test/e2e/responsive.spec.js` with a top-of-file import block and a `Phone` describe that later tasks (3,4,5,6,7,8,9) append tests to.

- [ ] **Step 1: Write the failing test** — create `test/e2e/responsive.spec.js`:

```js

import {bootApp, createTiddler} from './helpers.js';

const PHONE = {width: 390, height: 844}; // iPhone 12/13/14 logical size

test.describe('Phone (≤600px)', () => {
  test.use({viewport: PHONE});

  test('search results render as a full-width fixed overlay, not capped at the drawer width', async ({page}) => {
    await bootApp(page);
    await createTiddler(page, {title: 'OverlayProbe', text: 'find me overlay'});
    // Open the drawer so the search input is on-screen, then query.
    await page.evaluate(() => document.getElementById('sidebar').classList.add('open'));
    await page.fill('#search', 'OverlayProbe');
    await expect(page.locator('#search-results .tiddler-list').first()).toBeVisible();
    const r = await page.evaluate(() => {
      const el = document.getElementById('search-results');
      const cs = getComputedStyle(el);
      return {position: cs.position, width: el.getBoundingClientRect().width, vw: window.innerWidth};
    });
    expect(r.position).toBe('fixed');
    expect(r.width).toBeGreaterThan(r.vw - 2); // spans (essentially) the full viewport, > 280px
  });
});
```
